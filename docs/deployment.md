# Руководство по развертыванию Blazor Web App

## Поддерживаемые платформы

1. **GitHub Pages** - проект уже настроен для автоматического развертывания через GitHub Actions
2. **Статические хостинги** (Netlify, Vercel, и т.д.) - поддерживаются с минимальной настройкой
3. **Любой веб-сервер** - способный обслуживать статические файлы

## Развертывание на GitHub Pages

### Автоматическое развертывание через GitHub Actions

Проект уже настроен для автоматического развертывания на GitHub Pages через GitHub Actions. Workflow находится в файле `.github/workflows/deploy.yml`.

#### Настройка GitHub Pages

1. Создайте репозиторий на GitHub и отправьте в него код проекта
2. Перейдите в настройки репозитория (Settings → Pages)
3. В разделе "Source" выберите "GitHub Actions"
4. При следующем пуше в ветку `main` автоматически запустится workflow

#### Как это работает

1. При пуше в ветку `main` запускается GitHub Actions workflow
2. Workflow выполняет сборку проекта с помощью команды:
   ```
   dotnet publish src/Host/Host.csproj --configuration Release --output temp_deploy
   ```
3. Дополнительно выполняется подготовка файлов - копирование содержимого директорий `_framework` и `_content` в корень `wwwroot`, после чего оригинальные директории удаляются
4. Подготовленные файлы из директории `temp_deploy/wwwroot` публикуются на GitHub Pages

### Ручное развертывание

Если вы хотите вручную собрать и развернуть проект, выполните следующие шаги:

#### С использованием скриптов

Для Windows:
```cmd
cd src
build-and-deploy.bat
```

Для Linux/macOS:
```bash
cd src
chmod +x build-and-deploy.sh
./build-and-deploy.sh
```

После выполнения скриптов собранные файлы будут находиться в директории `src/temp_deploy/wwwroot`, которые можно загрузить на любой хостинг статических файлов.

#### Без использования скриптов

1. Соберите проект:
   ```bash
   dotnet publish src/Host/Host.csproj --configuration Release --output temp_deploy
   ```

2. Скопируйте содержимое директории `temp_deploy/wwwroot` на ваш хостинг статических файлов.

## Развертывание на других платформах

### Netlify

1. Создайте аккаунт на [Netlify](https://www.netlify.com/)
2. Подключите ваш репозиторий GitHub
3. В настройках сборки укажите:
   - Build command: `dotnet publish src/Host/Host.csproj --configuration Release --output temp_deploy`
   - Publish directory: `temp_deploy/wwwroot`
4. Запустите сборку

### Vercel

1. Создайте аккаунт на [Vercel](https://vercel.com/)
2. Подключите ваш репозиторий GitHub
3. В настройках сборки укажите:
   - Build Command: `dotnet publish src/Host/Host.csproj --configuration Release --output temp_deploy`
   - Output Directory: `temp_deploy/wwwroot`
4. Запустите сборку

### Статический хостинг

Для развертывания на любом статическом хостинге:

1. Соберите проект:
   ```bash
   dotnet publish src/Host/Host.csproj --configuration Release --output temp_deploy
   ```

2. Загрузите содержимое директории `temp_deploy/wwwroot` на ваш хостинг.

## Особенности развертывания Blazor WebAssembly

### Маршрутизация SPA

Для корректной работы маршрутизации одностраничных приложений на статических хостингах (включая GitHub Pages) используются специальные техники:

1. В корне проекта находится файл `404.html`, который перенаправляет запросы к несуществующим страницам обратно в приложение
2. В файле `index.html` и `App.razor` содержится JavaScript код для обработки маршрутов

### Оптимизация производительности

1. Все статические ресурсы (CSS, JS, изображения) имеют хэши в именах файлов для эффективного кэширования
2. Используется сжатие ресурсов (gzip/Brotli) при поддержке хостинга
3. Реализовано предварительное рендеринг для улучшения времени загрузки

## Устранение неполадок

### Проблемы с маршрутизацией

Если страницы не открываются при прямом переходе по URL:

1. Убедитесь, что файл `404.html` находится в корне вашего сайта
2. Проверьте, что JavaScript код для маршрутизации присутствует в `index.html`

### Проблемы с загрузкой ресурсов

Если стили или скрипты не загружаются:

1. Проверьте пути к файлам в `index.html`
2. Убедитесь, что все необходимые файлы были скопированы при сборке
3. Проверьте консоль браузера на наличие ошибок 404

### Проблемы с GitHub Pages

Если GitHub Pages не обновляется:

1. Проверьте статус последнего workflow в разделе Actions репозитория
2. Убедитесь, что в настройках Pages выбран источник "GitHub Actions"
3. Проверьте, что ветка `main` защищена от force-push
