# Обзор используемых библиотек

## Основные библиотеки проекта

### Microsoft.AspNetCore.Components.WebAssembly (v9.*)

**Назначение**: Основной фреймворк для создания Blazor WebAssembly приложений

**Функции**:
- Позволяет создавать интерактивные веб-приложения с использованием C# вместо JavaScript
- Обеспечивает выполнение .NET кода в браузере через WebAssembly
- Предоставляет компонентную модель для создания переиспользуемых UI элементов
- Включает маршрутизацию, управление состоянием и другие веб-функции

**Особенности в .NET 9**:
- Улучшенный пререндеринг страниц на сервере
- Оптимизированная загрузка и производительность
- Улучшенная интеграция с серверными компонентами

### MudBlazor (v8.*)

**Назначение**: Библиотека UI компонентов для Blazor, реализующая Material Design

**Функции**:
- Готовые компоненты интерфейса (кнопки, формы, таблицы, навигация и т.д.)
- Поддержка темной и светлой тем
- Адаптивный дизайн для различных устройств
- Расширенные возможности настройки внешнего вида

**Основные компоненты, используемые в проекте**:
- `MudText` - Текстовые элементы с различными типографическими стилями
- `MudButton` - Кнопки различных стилей и цветов
- `MudTable` - Таблицы с сортировкой и пагинацией
- `MudThemeProvider` - Компонент управления темами
- `MudAppBar`, `MudDrawer` - Компоненты навигации и макета
- `MudNavMenu`, `MudNavLink` - Элементы навигационного меню
- `MudAlert`, `MudProgressCircular` - Компоненты обратной связи

### Microsoft.AspNetCore.Components.WebAssembly.Server (v9.*)

**Назначение**: Серверные компоненты для поддержки Blazor WebAssembly приложений

**Функции**:
- Обслуживание клиентского Blazor WebAssembly приложения
- Поддержка пререндеринга страниц на сервере
- Обработка статических ресурсов
- Интеграция с серверными службами и API

## Вспомогательные библиотеки

### System и базовые библиотеки .NET

**Используются в проекте**:
- **System.Linq** - Для работы с коллекциями и выполнения запросов к данным
- **System.Net.Http** - Для выполнения HTTP запросов (в случае расширения функционала)
- **System.Collections.Generic** - Для работы со списками и словарями

## Архитектурные особенности использования библиотек

### Интеграция MudBlazor

Библиотека MudBlazor интегрирована в проект следующим образом:
1. В `Program.cs` клиентской части регистрируются сервисы MudBlazor:
   ```csharp
   builder.Services.AddMudServices();
   ```
2. В `MainLayout.razor` добавлены провайдеры компонентов:
   ```razor
   <MudThemeProvider />
   <MudPopoverProvider />
   <MudDialogProvider />
   <MudSnackbarProvider />
   ```

### Конфигурация темизации

MudBlazor позволяет гибко настраивать темы через:
- Определение собственных палитр цветов для светлой и темной тем
- Настройку свойств макета
- Переключение между темами в runtime

### Компонентная модель

Все используемые библиотеки поддерживают компонентную модель разработки:
- Переиспользуемые компоненты
- Однонаправленный поток данных
- Жизненный цикл компонентов
- Обработка событий
